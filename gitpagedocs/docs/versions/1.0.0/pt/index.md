# Casecell Checkout Service

Bem-vindo à documentação oficial do projeto entregue no **Desafio Técnico CaseCellShop — Nível Pleno (Fullstack)**. Aqui você encontra a visão completa da solução, desde a motivação do case até detalhes de arquitetura, APIs, experiência do usuário e estratégias de qualidade.

## Como navegar

- **Visão geral do case**: contextualiza os problemas reais da CaseCellShop e como o projeto os resolve.
- **Arquitetura e design**: descreve a separação de responsabilidades entre frontend, backend, banco, cache e simulador de ERP.
- **Back-end**: cobre endpoints, validações, idempotência e simulação de instabilidades.
- **Front-end**: mostra como a vitrine, carrinho e checkout garantem UX responsiva e resiliente.
- **Qualidade & operações**: traz os comandos de execução, variáveis de ambiente, Docker e observabilidade.
- **Perguntas conceituais (Parte 1.A)**: registra as respostas exigidas no desafio.

> ℹ️ Todo o conteúdo está disponível em **português** e **inglês**. Use o seletor de idioma para alternar entre as versões.

## Repositório público

O código-fonte está disponível em: <https://github.com/Vidigal-code/casecell-checkout-service>

## Resumo rápido

- **Stack principal**: NestJS, Prisma, PostgreSQL, Redis, BullMQ, Next.js 15, React Query, Tailwind CSS.
- **Objetivo**: construir um fluxo de checkout resiliente, desacoplado do ERP, prevenindo overselling e oferecendo UX confiável.
- **Entrega**: APIs completas, frontend funcional, documentação bilíngue, testes automatizados e scripts Docker.
